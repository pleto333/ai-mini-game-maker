# AI Mini Game Maker 발표용 요구사항 및 데모 시나리오

## 1. 프로젝트 목적

사용자가 자연어로 미니 게임 아이디어를 입력하면 AI가 즉시 실행 가능한 HTML5 게임을 생성하고, 이후 대화형 수정 요청을 통해 게임을 계속 고도화할 수 있는 웹 기반 제작 도구를 만든다.

## 2. 핵심 요구사항

### 기능 요구사항

- 사용자는 만들고 싶은 게임을 자연어로 입력할 수 있어야 한다.
- 최초 요청 시 AI는 CSS와 JavaScript가 포함된 단일 HTML 문서를 생성해야 한다.
- 추가 요청 시 기존 게임 코드를 바탕으로 전체 HTML을 다시 생성해야 한다.
- 생성된 게임은 브라우저에서 즉시 플레이 가능해야 한다.
- 사용자는 게임 플레이 화면과 전체 소스코드를 전환해서 볼 수 있어야 한다.
- 사용자는 게임을 안전하게 리셋할 수 있어야 한다.
- 사용자는 완성된 게임을 HTML 파일로 내보낼 수 있어야 한다.
- 사용자는 이미지 에셋을 업로드하고 역할을 지정할 수 있어야 한다.
- 사용자는 이전 생성 버전을 히스토리에서 복구할 수 있어야 한다.
- 사용자는 프로젝트 상태를 JSON으로 내보내고 다시 불러올 수 있어야 한다.

### 비기능 요구사항

- API 키는 프론트엔드에 노출하지 않고 서버 환경변수로 관리해야 한다.
- AI 호출 실패 시 입력값과 대화 흐름이 유실되지 않아야 한다.
- iframe 내부에서 `location.reload()`를 쓰지 않고 부모 앱이 코드를 재주입해야 한다.
- Gemini와 OpenAI를 교체할 수 있는 구조여야 한다.
- Render 같은 Node 배포 환경에서 바로 실행 가능해야 한다.
- 무료티어 사용을 고려해 호출 횟수를 줄일 수 있어야 한다.

## 3. 설계 및 아키텍처 근거

- **프론트/백엔드 분리**: Vue는 UI와 상태를 담당하고, Express는 API 키와 AI 호출을 담당한다.
- **프롬프트 빌더 패턴**: 최초 생성, 수정, 이미지 에셋, 메타데이터 요청을 함수 단위로 분리했다.
- **Provider Adapter 구조**: `AI_PROVIDER` 값에 따라 Gemini/OpenAI 호출 함수를 선택한다.
- **샌드박스 렌더링**: AI가 생성한 HTML은 iframe에 주입해 메인 앱과 분리한다.
- **부모 제어형 리셋**: iframe reload 대신 Vue가 저장된 HTML을 다시 주입한다.
- **버전 히스토리**: 생성 결과를 버전 단위로 저장해 롤백과 비교가 가능하다.
- **에셋 placeholder 방식**: 이미지 data URL을 직접 프롬프트에 길게 넣지 않고 placeholder로 치환한 뒤 응답 후 삽입한다.
- **장애 복구 UX**: 렌더링 실패 감지, 자동 수리, API 오류 시 입력 복구를 제공한다.

## 4. 발표 데모용 예시 게임

### 예시 1: 우주선 피하기 게임

초기 요청:

```text
방향키로 우주선을 움직이고, 위에서 내려오는 운석을 피하는 HTML5 미니 게임을 만들어줘.
점수는 생존 시간에 따라 올라가고 충돌하면 게임오버 화면이 나오게 해줘.
```

수정 요청:

```text
20초마다 운석 속도와 개수를 늘려서 난이도가 올라가게 해줘.
게임오버 후 다시 시작 버튼도 크게 보여줘.
```

보여줄 포인트:

- 최초 생성과 추가 수정의 차이
- 난이도/밸런스 조정 모드
- 안전 리셋과 HTML 내보내기

### 예시 2: 점프 러너 게임

초기 요청:

```text
스페이스바로 점프해서 장애물을 피하는 러너 게임을 만들어줘.
배경은 밤 도시 느낌이고 점수와 최고 점수를 보여줘.
```

수정 요청:

```text
이중 점프 기능을 추가하고 장애물 종류를 2개로 늘려줘.
모바일에서도 버튼으로 점프할 수 있게 해줘.
```

보여줄 포인트:

- 모바일 최적화 수정
- 기능 추가 모드
- 버전 히스토리 복구

### 예시 3: 이미지 캐릭터 게임

초기 요청:

```text
업로드한 이미지를 플레이어 캐릭터로 사용해서 코인을 모으는 게임을 만들어줘.
적에게 닿으면 체력이 줄고, 코인을 10개 모으면 승리하게 해줘.
```

수정 요청:

```text
배경 이미지를 더 크게 보여주고, 코인 획득 시 반짝이는 효과를 넣어줘.
UI는 오른쪽 위에 작게 정리해줘.
```

보여줄 포인트:

- 이미지 에셋 역할 지정
- 멀티모달 확장성
- 디자인 개선 요청

## 5. 평가 항목 대응 논리

- **완성도**: 실제 배포 URL, 빌드 통과, 게임 생성/수정/실행/저장 흐름으로 증명한다.
- **아이디어 구체화**: 단순 생성기가 아니라 프롬프트 분기, 수정 모드, 이미지 에셋, 히스토리, JSON 백업까지 설계했다.
- **창의성**: AI 코드 생성 결과를 바로 플레이하고 다시 대화로 개선하는 제작 루프가 차별점이다.
- **지속 가능성**: provider 교체, Render 배포, GitHub 공개, 백업/복원 구조로 계속 발전 가능하다.
- **AI 활용도**: Codex/ChatGPT는 기획·구현·디버깅·문서화에, Gemini/OpenAI는 실시간 게임 생성에 활용했다.
